Example of a single node (leaf) JSON response:

Inside of code Tree Data is JavaScript Object. Therefore tree leafs are simply fields of that object.
So, there is strong naming convention used for storing data about leafs: 


{
	// &ndash; inside of current branch must be unique name, used as tree object property inside of code,
	'unique_naming_string' : {

		// &ndash; optional, that will be user for presentation instead of .name,
		text : 'string',

		// &ndash; optional boolean, that indicates leaf is folder or not, "folder" means there are children inside,
		folder : [true || false],

		// &ndash; optional boolean, indicates leaf children must be loaded and leaf must be open during rendering
		open : [true || false]
		
		// and all that You need could be used as other leaf properties
		
	}
}


Config Explanation {
  • root – name of root branch, must be used!
  • init – 

// parameters used while init and loading root branch
init : {
	
	// delay before loading root in miliseconds, or null
	delay : null,
	
	// class name, used for tree container while loading rood branch
	// case not provided, there will be no preloader
	preloader : 'preloader',
	
	// function (controller, tree) used imediately after init
	callback : null,
	
	// method user for expanding rood branch
	method : 'fadeIn',
	
	// make auto init, means tree component will load itself without any other options
	auto : true,
	
	// path of the leaf for initial focus()
	focus : null

},


  • handlers ( leaf, controller, tree ) – a set of pre-configured event handlers that will be used while manipulation with tree.

    Each callback could use standard pack of arguments:

    leaf – current leaf for callback

    controller – tree controller, used for API manipulation

    leaf – tree Object itelf.
    THIS IS NOT A LOADED DATA, this is object that was created for the loaded data!
    Means it contains all of [leafs] passed here.

    but 'beforeblur' handler uses another set:

    ( callback, leaf, controller, tree )

    Here the Callback is function, you must call when you need this leaf blured.
    If leaf can't be blured right now, just don't use this callback.


  • loader ( path, callback ) – you must to write this function.

    It will be used by a tree controller as a single way to load data about tree root and tree leaf childrens.

    It receives path array :

    path = ['tree.root', 'leaf.name']


    and callback function, where you must to pass JS Object of loaded childrens for this path.
    It must be not JSON, it must be an object!

  • listeners – event handlers in jQuery .on(...) format, that would be used as callbacks while user manipulations with rendered leaf.text.

    For example, if you wish to drag item, or use context menu of item, or just need one else 'click' callback instead of 'focus' preset -- use this option.

  • theme – CSS class name PREFIX_ for tree container class name.

  • cls – set of CSS class names for rendering: 

cls : {

	// tree root class name
	root : 'tree_root',

	// tree controll class name, is there where plus and minuse indicates folder is open or not
	control : 'tree_control',
	
	// status used for indication that leaf is loading, if you need class name there -- use this option
	status : 'tree_leaf_status',
	
	// leaf text is name for symbols of leaf, this is what is written on leaf
	text : 'tree_leaf_text',

	// if you need optional folder class name
	folder : 'folder',
	
	// this will be used when leaf is selected, when .focus() handles
	selected : 'selected',
	
	// when mouse is over leaf text
	hover : 'hover',

	// when leaf is loading, used for 'status'
	loader : 'loader',
	
	// used for indicating open leaf
	open : 'open',

	// tree container class name
	container : 'container',

	// when you don't whant to use tree text selection
	supressLabelTextSelection : 'unselectable',
	supressTreeTextSelection : 'unselectable'

}

  • html – what HTML tag names you will use for rendering: 

html : {

	// tree root container tagName
	tree : '<UL>',
	
	// leaf container tagName
	leaf : '<LI>',
	
	// where children stored inside of leaf
	children : '<UL>',

	// what you will use for leaf text and control elements
	heading : '<DIV>',
	
	// there wher + & - are
	control : '<SPAN>',
	
	// there wher current status and 'loading' caption indicates
	status : '<SPAN>',
	
	// there where leaf.text is
	text : '<SPAN>',

	// whole container for tree container
	container : '<DIV>'

},

  • control – what strings you will use for rendering controls "plus and minus": 

control : {
	close : '+',
	open : '&ndash;'
},

  • storeLoaded: true || false – tree controller could store loaded leaf for reusing when open\close, or just load them each time
  • focusParentOnClose: true || false – tree controller could focus parent element when closing focused child
  • focusByDblClick: true || false – instead of focusing by click you could with to focus by "double click"
  • blurFromContainerClick: true || false – you could wish to blur current focused leaf when clicks on tree container
  • blurFromContainerDblClick: true || false – simply as previous but for double click
  • labelsBreak: – you could wish to use extremely lont texts for leafs, here is options where you could configure rendering 

labelsBreak : {

	// how many symbols to break text string by
	by : 0,
	
	// with what you wish to break 
	str : '\n',
	
	// if you wish whole string is always shown
	expandAlways : false,
	
	// if shown only when hover
	expandOnHover : false,
	
	// if shown only when focused
	expandOnSelect : true
}



}

Controller API

  • getPath ( leaf ) – builds URL ARRAY from Leaf Object: [ 'top', 'leaf_name', 'child_name' ... ]
  • refresh ( leaf, callback, andOpen ) – reloads children of passed 'leaf',
    'callback' will be fired after completition,
    pass 'andOpen' as true if you need to re-open closed leaf
  • blur ( leaf ) – tries to blur passed leaf, look beforeblur handler for understanding
  • focus ( leaf ) – tries to focus passed leaf, look beforeblur handler for understanding
  • x – current controller config described above
  • x.current – current focused leaf
  • init – if config.init.auto == false, here would be 'manual' init function for call when necessary
